# Java Design Patterns

Repositório dedicado ao estudo e implementação dos principais padrões de projeto em Java, focado em criar códigos mais flexíveis, reutilizáveis e fáceis de manter.

## 📖 O que é um padrão de projeto?

Padrões de projeto são soluções conceituais para problemas recorrentes no desenvolvimento de software. Diferente de algoritmos, que fornecem passos exatos, os padrões funcionam como "plantas de obra" personalizáveis, focadas em alto nível para resolver questões de estrutura.

*   **Padrão vs Algoritmo:** Enquanto algoritmos são receitas com etapas claras, padrões funcionam mais como plantas de construção, onde a implementação final depende do contexto do seu projeto.

> Documentação ref.: [Refactoring.Guru/what-is-pattern](https://refactoring.guru/pt-br/design-patterns/what-is-pattern)

## 🏗️ Catálogo de Padrões

O conteúdo está organizado conforme a classificação clássica, com exemplos práticos baseados na estrutura:

### 1. Padrões Criacionais (`src/Creation/`)
Mecanismos de criação de objetos para maior flexibilidade:
*   *Factory Method, Abstract Factory, Builder, Prototype, Singleton.*

### 2. Padrões Estruturais (`src/Structural/`)
Organização de classes e objetos em estruturas maiores:
*   *Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy.*

### 3. Padrões Comportamentais (`src/Behavioural/`)
Gestão de responsabilidades e comunicação entre objetos:
*   *Chain of Responsibility, Command, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor.*

> Catálogo ref.: [Refactoring.Guru/catalog](https://refactoring.guru/pt-br/design-patterns/catalog)

## 📝 Do que consiste um padrão?

A documentação dos padrões neste repositório aborda os seguintes pontos, essenciais para sua compreensão e aplicação:

*   **Propósito:** Breve descrição do problema e solução.
*   **Motivação:** Cenários onde o padrão se aplica.
*   **Estruturas de classes:** Relacionamento entre as partes.
*   **Exemplos de código:** Implementações práticas em Java.
